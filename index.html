Responsive Redesign

Redesigning a local Greek Orthodox church website into a beautiful responsive website!

Identify Usability Problems:

The images below show the home page of the Anunciation Greek Orthodox Church in Cranston, RI. At first glance, the main page does not seem to contain many problems, but as follow along this site, you will uncover some design flaws. We will go over those problems and offer up a potential redesign of the website!

Click the button below to explore the site yourself!

Now, hover over the icons below to uncover usability problems such as the learnability, memorability, and accesibility of the website!

Learnability

How easily can new users learn to use the website?

  1. There is no clear sequence for the user to follow on the home page.
  2. Many images are linked to another page, but this is not made very clear to the user.
  3. Some words and phrases used are unclear as to what they mean and there is no indication for their definition or significane.

Memorability

Do users remember how to use the website when coming back to it?

  1. Someone looking for schedule of services would have difficulty remembering how to find what days services are as that is not made clear on the home page.
  2. The menu bar has a lot of items on it, so users can forget what each one is meant for.

Accesibility

Is the website accessible to users with different needs (i.e. visually impairment)?

  1. Lots of images and linked images are missing alt tags.
  2. Very low contrast between the text and background colors.
  3. Several empty headings are found throughout the website.

Visual Redesign:

Now that we have identified some design problems in this website, let's use all that information to guide us as we redesign the website!

Using Balsamiq, we can make some low-fidelity prototypes. Below you will see a low-fidelity prototype for our redesign on a mobile device, tablet, and a desktop.

These low-fidelity prototypes effectively solve many of the problems that users may encounter. One notable issue was a lengthy amount of menu tabs which can easily confuse a user trying to navigate the page efficiently. I found that many of that information can be transferred onto the home page to make better use of space while still maintaining a legible interface. Additionally, users can follow a more clear sequence on the page. Essentially, instead of all the information being cluttered, it is now simplified to blocks of information with clear divisions.

Before we begin making high-fidelity prototypes, let's make a visual design style guide so that our design is consistent across the board. This can be done using Figma!

Upon looking at this visual design guide, you might notice that the colors selected are very different than those in the original website. This is done on purpose because I believe a more clear and modern color scheme will help with the overall look and feel of the site. On the original site, there was no clear color palette so this will definitely make the site more visually pleasing. Blue can be a very friendly and welcoming color which will help this site serve its purpose to engage with new users and get more people involved with them.

Let's now use our low-fidelity prototypes and visual design style guide to create some high-fidelity prototypes for mobile, tablet, and desktop.

Here we can see that the color palettes, typography, and buttons from the style guide have been used effectively here to create the high-fidelity prototypes. However, one important thing we need to ensure before we start to code our design, is how to make our website responsive. A responsive website is one that automatically changes size in conjunction to the screen size changing.

This will be accomplished in many ways. Firstly, we will use flexbox primarily because of its ability to be dynamic and that will be useful as we play around with different screen sizes.

Another thing to be aware of is that our text may change size as we change screen sizes. We can adjust this by using 'media query' to tell the website to change font and text sizes dependent on what the desired screen size is.
